# 2026-05-22 - MV Popup Manager v2.8.3 Bug Fixes

## Version 2.8.3 - Template System Bug Fixes

**Status:** ✅ Production Deployed

### Overview

Bug fix release addressing field checkbox persistence, phone field functionality, and template consistency issues discovered after v2.8.0 template system deployment.

### Bug Fixes Completed

#### 1. Field Checkboxes Now Persistent
**Issue:** When switching templates, the "Include Phone Number" checkbox would uncheck itself.

**Root Cause:** The `handleTemplateSelect` function was resetting `includeFirstName` and `includePhone` based on each template's default fields configuration.

**Fix:** Removed the lines that reset these values when switching templates.

**File:** `app/admin/popup/edit/page.js`

#### 2. Phone Field Save/Load Fixed
**Issue:** Phone field checkbox state wasn't being saved to the database or loaded back.

**Root Cause:** The save API was only saving `firstName` and `email` fields, ignoring `phone`.

**Fix:** Updated the save route to include phone in the fields array.

**File:** `app/api/popups/save/route.js`

#### 3. Ultra Minimal Duplicate Email Fixed
**Issue:** Ultra Minimal template showed two email fields.

**Fix:** Removed the duplicate email input.

**File:** `app/admin/components/templates/PopupPreview.js`

#### 4. Single Name Field Consistency
**Issue:** Some templates showed separate "First Name" and "Last Name" fields.

**Fix:** Changed all templates to use single "Name" field.

**Files:** `app/admin/components/templates/PopupPreview.js`

#### 5. Phone Field Added to All Templates
**Issue:** Several templates were missing the phone number field.

**Fix:** Added phone field to Ultra Minimal, Lead Magnet, Split Lead Magnet, Personal Consultation, and Full Background templates.

#### 6. Name Field Added to Missing Templates
**Issue:** Several templates were missing the Name field option.

**Fix:** Added Name field to all templates that were missing it.
