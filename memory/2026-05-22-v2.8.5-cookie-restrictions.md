# MV Popup Manager v2.8.5 - Cookie-Based Popup Restrictions

**Date:** 2026-05-22  
**Version:** 2.8.5  
**Status:** Production

## Overview

Version 2.8.5 introduces cookie-based restrictions for exit intent and time delay popups, while ensuring button popups remain unrestricted for maximum conversion optimization.

## Cookie Behavior

### Button Popups
- **NO cookie restriction**
- Always show when button is clicked
- Critical for conversion optimization (visitors may not opt-in on first visit but return later)

### Exit Intent Popups
- **7-day cookie restriction**
- Shows once per popup ID per 7 days
- Cookie name: `gc_popup_{popupId}`
- Triggers when mouse moves to top of viewport (e.clientY < 10)

### Time Delay Popups
- **7-day cookie restriction**
- Shows once per popup ID per 7 days
- Cookie name: `gc_popup_{popupId}`
- Default delay: 3 minutes (180 seconds)

## Technical Implementation

### Frontend Library (gc-modal.js)
- Version: 2.8.5
- Cookie expiry: 7 days
- API endpoint: https://gcmodal-api.vercel.app

### Key Code Changes
1. `setupDirectTrigger()` - Handles admin-generated code with popupId and trigger
2. `hasSeenPopup()` - Checks for existing cookie
3. `setPopupSeen()` - Sets 7-day expiry cookie
4. `initUniversal()` - Backward compatibility alias for existing implementations
