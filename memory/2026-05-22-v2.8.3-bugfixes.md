# MV Popup Manager v2.8.3 - Template System Bug Fixes

**Date:** 2026-05-22  
**Version:** 2.8.3  
**Status:** Production

## Overview

Bug fix release addressing field checkbox persistence, phone field functionality, and template consistency issues discovered after v2.8.0 deployment.

## Bug Fixes

### 1. Field Checkboxes Now Persistent (v2.8.3)
**Issue:** When switching templates, the "Include Phone Number" checkbox would uncheck itself.

**Root Cause:** The `handleTemplateSelect` function was resetting `includeFirstName` and `includePhone` based on each template's default fields configuration.

**Fix:** Removed the lines that reset these values when switching templates. Now only `template`, `layout`, and `variant` change - field checkboxes persist.

**Code Change:**
```javascript
// Before (in handleTemplateSelect):
setPopup(prev => ({
  ...prev,
  template: template.id,
  layout: template.config.layout,
  variant: template.config.defaultVariant,
  includeFirstName: template.config.fields.includes('firstName'),  // ❌ Resetting
  includePhone: template.config.fields.includes('phone')           // ❌ Resetting
}))

// After:
setPopup(prev => ({
  ...prev,
  template: template.id,
  layout: template.config.layout,
  variant: template.config.defaultVariant
  // Field checkboxes now persist!
}))
```

### 2. Phone Field Save/Load Fixed (v2.8.3)
**Issue:** Phone field checkbox state wasn't being saved to the database or loaded back.

**Root Cause:** The save API was only saving `firstName` and `email` fields, ignoring `phone`.

**Fix:** Updated the save route to include phone in the fields array when `includePhone` is true.

**Code Change:**
```javascript
// Before:
fields: popup.includeFirstName ? ['firstName', 'email'] : ['email']

// After:
fields: [
  ...(popup.includeFirstName ? ['firstName'] : []),
  'email',
  ...(popup.includePhone ? ['phone'] : [])
]
```

### 3. Ultra Minimal Duplicate Email Fixed (v2.8.3)
**Issue:** Ultra Minimal template showed two email fields - one at the top and one after the Name field.

**Fix:** Removed the duplicate email input that appeared before the Name field.

### 4. Single Name Field Consistency (v2.8.0-2.8.3)
**Issue:** Some templates showed separate "First Name" and "Last Name" fields while others showed single "Name" field.

**Fix:** Changed all templates to use single "Name" field for consistency:
- Clean Gradient: Changed from First/Last split to single Name
- Split Screen: Changed from First/Last split to single Name
- All other templates: Already had single Name field

### 5. Phone Field Added to All Templates (v2.8.0-2.8.3)
**Issue:** Several templates were missing the phone number field entirely.

**Fix:** Added phone field to:
- Ultra Minimal
- Lead Magnet
- Split Lead Magnet (mobile & desktop)
- Personal Consultation (mobile & desktop)
- Full Background (mobile & desktop)

### 6. Name Field Added to Missing Templates (v2.8.0-2.8.3)
**Issue:** Several templates were missing the Name field option.

**Fix:** Added Name field to:
- Ultra Minimal
- Lead Magnet
- Split Lead Magnet (mobile & desktop)
- Personal Consultation (mobile & desktop)
- Full Background (mobile & desktop)
